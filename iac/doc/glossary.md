# Glossary

Alphabetized list of terms used in the Multi-Cloud IAC Framework.

---

## A

**API Layer**
: The third layer of the SEA architecture, defining provider-agnostic resource contracts (inputs/output).

**AWS (Amazon Web Services)**
: One of the supported cloud providers.

**Azure (Microsoft Azure)**
: One of the supported cloud providers.

---

## C

**Common Layer**
: The first layer of the SEA architecture, containing shared types, constants, and normalization logic.

**Core Layer**
: The fourth layer of the SEA architecture, responsible for resource orchestration and dependency management.

---

## F

**Facade Layer**
: The fifth and public layer of the SEA architecture, providing a unified and simplified interface for infrastructure deployment.

---

## G

**GCP (Google Cloud Platform)**
: One of the supported cloud providers.

---

## I

**IAC (Infrastructure as Code)**
: The practice of managing and provisioning computing infrastructure through machine-readable definition files.

---

## S

**SEA (Stratified Encapsulation Architecture)**
: The 5-layer architectural pattern used in this project, mirrored from the CloudKit SDK.

**SPI (Service Provider Interface) Layer**
: The second layer of the SEA architecture, handling provider authentication, session management, and remote state.

---

## T

**Terratest**
: A Go library that makes it easier to write automated tests for your infrastructure code.
