## Configuration Files

### context.yaml

Defines all contexts (provider, role, environment) and workflows.

### routes.yaml

Context-aware routing with dynamic parameters.

### nav.yaml

Context-aware navigation with role-based filtering.

---

## Core Concepts

### Contexts

1. **Provider Context** - Which cloud (AWS, Azure, GCP, Zero)
2. **Role Context** - User permissions (Developer, DevOps, Admin)
3. **Environment Context** - Target environment (Local, Dev, Staging, Prod)
4. **Workflow Context** - Active workflow state

### Workflows

1. **create-resource** - Create cloud resources (S3, DynamoDB, etc.)
2. **deploy-infrastructure** - Terraform plan/apply flow
3. **debug-request** - Debug and replay requests
